写下此篇流水纯粹是为了重拾那些零星的记忆，只能建立起一个模糊的脉络，找不回具体的时间点了，现在也是试着能否从某些日志里寻回些线索来。本篇日志最早创建于 2014 年 2 月份，中间偶有修改，却一直在草稿箱里静静的躺着。终于又时隔多年首次放出一篇非技术文出来。

1. 开始的开始，2001 年工作起，进入一个几乎完全陌生的程序世界。为了奋力从大海洋中汲取营养，大跃进的方式学习，并坚持在笔记本上面现在看来像是抄写一样的记笔记，不觉几年功夫，工工整整，满满的记载了四五本笔记本。还有看到那些好的东西也爱打印下来，其实打印出来的东西与它在网络上是一回事，没完整看的还是没看完，写下来了可不一样。这样记笔记的习惯其实也是工作之后的事情，上学期间最爱在课本上记东西。

2. 到后来是网络的盛行，也是信息量的爆炸的时代，再系统再全的书籍也无法与 Google 相比了。所以大概是 2005 年想到了在 QQ 空间里记了三五篇，一块娱乐之地上记录这些东西格调十分不搭。

3. 找到了第一个真正意义上的 BSP，那就是 blogcn.com，它现在完全变样了。QQ 上的几篇只需要复制到编辑框保存就行，那应该是  2006 年的事情。blogcn 是个综合性的 BSP，不好玩，代码高亮用工具保存成的带样式的 HTML 代码，好像也只在这里折腾了一年多时光，到  2007 吧，继续寻租。

4. 寻到 blogjava.net，总算是专业，特别是对于以 Java 为主的人员。搬迁首次自动化了，自己写程序把 blogcn.com 上的文章生成 rss.xml，通过 blogjava.net  的 rss 导入功能来完成，也难免有些修修补补的。其实中间有考虑弄到 javaeye 上去的想法。终于进到专业的圈子里了，技术交流氛围也不错。从一些日志文件的日期来看应该是在 blogjava.net 上度过了 2007-2010 年。

5. 放在别人家的东西多了，还是希望能自成一家，于是动了做成独立博客的念头，再一个原因是在现今的公司学到了 WordPress 的高级应用。既然不愿备案，只能到外面看看。于 2010 年 7 月份跑到 bloghost.cn 上买了首个域名(unmi.cc) 和空间，搭建了自己的 WordPress，同时也确立了这一主旋律。于是再次把所有日志生成一个个 rss 文件，应用 WordPress  的 rss 导入。这回把原先在 blogjava.net 上日志评论也搬过来了，当然也有手动加工的少量工作。

6. 在美国的主机上购买的空间是在 2010-07 到  2012-07 这个区间内，实际使用也就一年多时间吧。由于国内访问实在是太慢了，有时候自己写着写着东西都要保存--丢失连接--又一次凭着记忆重输一遍。

7. 为求速度， 2012-04 月份在壹互联买了香港的主机，速度体验上还不错，就是服务器烂，隔三岔五的出问题。终于 2013-12 月份那台服务器彻底歇菜了，他们拙劣的备份技术没救回文件。幸好中间有过一次内部换服务器，才捡回了 2012-09 月份之前的文件数据，加上自己每周的数据库备份，才不致使得前功尽弃，欲哭无泪。

8. 香港没法呆了，主机不够隐定，索些弄个 VPS 来玩，在 2013-12 迁移到我现在所在城市芝加哥的 VPS 上，装的是 Debian6+Apache，自由度很大。就还是老问题，国内访问速度慢，我自己写博客倒是爽。

9. 速度还是速度，2014-02-11 把博客安装到新加坡的亚马逊云服务器上了，趁着亚马逊有一年的免费期，不过得小心翼翼的使用，免费是各种条款的。使用了 EC2 和 RDS 这两个服务，EC2 就是 VPS，安装的是 Debian6+Nginx，之后的任务就是紧紧的管好自己在亚马逊的帐单。后来发现我装的 Nginx 没有传说中的那么好，估计是没配置好，所以又换回 Apache 了。

10. 在 AWS 的新加坡 VPS 上，从美国经常访问有问题，加上自己弄 VPS，本人技术上还未真正过硬。2014-02-18 日有点贱贱的又回到了香港壹互联的主机上了，不得不承认机房里的东西要稳定些，为弥补之前数据的损失，他们给了一年的免费期。

11. 香港基本是沦陷了，加之需要更大的操作自由度，仍然需要一个 VPS。又得考虑国内访问速度，偶尔寻见 ConoHa 日本的 VPS 服务，价钱还算公道，比 Linod 还是要划算许多。于是 2015-06 月份再次搬家到日本的 VPS 去了，时至今日，一切顺当，并无大事。

最后的最后，希望不用再往下记录了。或者是有朝一日，如果没有令人难堪的手续的话或许还会把博客搬入国内 VPS 的，为速度考虑。又或是网络真正成了一个地球村后，站点放哪儿都不会是个问题。
