通过我们在 General Settings 里都是把 WordPress address (URL) 和 Site address (URL) 设置成一样的，但它们可以不一样的。比如说 wordpress 整个目录传到了虚拟主机中，默认的你要用 http://unmi.cc/wordpress 的方式来访问，这当然不太好看，你希望用 http://unmi.cc 的 URL 来访问，可以用的做法是把 wordpress 下的文件放在虚拟主机根目录中。

另一种做法是仍然让保留 wordpress 目录，把其中的 index.php 以及 .htaccess 两个文移到根目录下，再编辑这个 index.php 文件，把其中的

require('./wp-blog-header.php');     改为

require('./wordpress/wp-blog-header.php'); 即可，

再到 General Settings 里把 WordPress address (URL) 设置为 http://unmi.cc/wordpress，Site address (URL) 设为 http://unmi.cc 就好了，用 http://unmi.cc 访问的就是 http://unmi.cc/wordpress 中的内容了。这可参考：http://codex.wordpress.org/Giving_WordPress_Its_Own_Directory。

再就是永久链接，你肯定不想总是用 http://unmi.cc/?p=123 的方式来访问你的日志，那就在 Settings/Permalinks 里设置一个你喜欢的。永久链接就是一种伪静态实现，最好能带上 %postname%，这样 URL 能够表意，在看访问统计时，一看到 URL 就知道是哪篇文章倍受关注，用 123.html 就看不太明白的。还有得提请注意的是，我先前也与人讨论过自定义 URL 后端是加“ /” 还是不加“ /”，也就是是用“ http://unmi.cc/blog-wordpress-book-reading/” 还是用 “http://unmi.cc/blog-wordpress-book-reading”，其实他们是有差别的，不加 "/" 的话还会多一次 301 跳转，所以建议最后加个 "/"，你看 WordPress 的插件页面链接都有" /" 的。

需要顾名思意的了解主题目录中各文件的职责，如 404.php、archive.php、comments.php、comments-popup.php、footer.php、functions.php、header.php、images.php、index.php、page.php、search.php、sidebar.php、single.php、 style.css 等。不一定每个主题都有所有的这些文件，基本上如果你想修改代码来实现某些插件的功能，主题中多数文件你都会去碰。
