--- title: Xcode4 相较于 Xcode3 使用中一些体验 url: /xcode4-some-experiences/ date: 2011-06-18T12:10:51-05:00 featured: false draft: true toc: false # menu: main usePageBundles: true categories: - iOS tags: - ios - xcode comment: true codeMaxLines: 50 # additional wpPostId: 3660 wpStatus: publish views: 843 lastmod: 2013-03-28T03:20:21-05:00 --- 从 Xcode3 升级到 Xcode4 的一些使用体验：

1. 注释的时候更智能，按 Command ＋ / 时不会像 Xcode3 里那样在光标处插入 //，而是会在行首加 //，并且多行注释时 // 很整齐了

2. 大括号有自动补齐的功能了，方法后输入左花括号 { 后，右边的花括号 } 由 Xcode 自动加上。

3. 智能提示中的内容丰富的多，方法的参数返回类型看得很清楚，尚不明最前面标识的 K、f、P、V、x、T、C、M、＃等是什么意义，不过也差不多有些概念了。

4. @interface 的模板也不一样了，出现默认 @private 修饰关键字，这是对的，默认的实例变量应该是隐藏的，@interface 的模板是：

@interface <# class name #> : <# superclass #> {
@private
<# instance variables #>
}
@end

5. 进到当前类或方法的实现，不像以前用 Command 变成十字时，双击某个类或方法，而有点像多数 IDE 那样，按下 Command，相应的类、方法或实例名变成链接那样，光标手型文字带下划时，这时个谁都知道可以去点击它一下进到实现里去。

6. 在 .h 和 .m 间切换，以前的 Option + Command + 上箭头，现在变成了 Control + Command + 上/下箭头。Option 替代成 Control，且原来只能用上箭头，如今可上可下。如果你用 Touchpad 的，三个手指上下扫可在 .h 和 .m 文件间轮换。
