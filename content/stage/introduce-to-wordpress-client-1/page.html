其实一直以来，我写博客都比较喜欢在线来写，省得装那些客户端，主要是曾经试用 Windows Live Writer 写出的东西垃圾代码太多，且文件上传时目录不好控制。不能因为一次不愉快的经历就害怕十年吧。为什么又再次提起博客的离线客户端呢？因为现在风声吃紧，特别是对于使用国外空间的，你非常的谨慎的写着，该避讳的地方也避了，还是不知哪个字成敏感词，一提交，网络被重置，完蛋了，抓逛也无济于事，一切重来，如果你没有事先 Ctrl + C 的话。所以我现在网上养成了一个良好习惯，写一段话在提交之前必按 Ctrl + C。

找个博客的客户端程序是十分必要的，由于我的是用的 WordPress，所以在此介绍几款支持 WordPress 的客户端。有了 WordPress 客户端不光可以不受胡萝卜的侵扰，还能让你无论何时何地自由的书写，连上线后即发布，上传图片文件等事也省了许多麻烦，还有更多的在线编辑不能提供的功能。

在 WordPress 的官方网站列出了各方平台下可用的 WordPress 客户端，见链接 http://codex.wordpress.org/Weblog_Client，肯定使用 Windows 桌面的居多数，所以还是先体验下 Windows 下的那些工具。等我某年某月的某一天买来了 Mac 后再说 Mac 的客户端。在该链接中列出了以下 Windows 的客户端：

  • BlogDesk - supports German and English
  • BlogJet
  • WordPress Comments notifier - open source (它只是一个评论提醒的客户端，日评论数不多的网站上后台看就行)
  • Chrysanth WebStory - 90-day trial
  • Ecto - 21-day trial (不知道谁在骗我，只找到 Mac 版本的)
  • LIPIDr Blog Client - freeware, last version: 2007 (.Net 程序，两文件，总尽寸 201K，配置时总出现程序错误，不能用)
  • Microsoft Word 2007
  • Post2Blog - detailed WordPress instructions
  • QTM
  • Qumana
  • RocketPost
  • Semagic
  • Zoundry Raven - open source
  • w.bloggar - freeware, last version: 2007
  • WB Editor
  • Windows Live Writer

1. BlogDesk 星级 2.5

当前版本 2.8，下载链接：http://www.blogdesk.org/en/blogdesk-284-en.exe, 大小 3.8M，它支持 WordPress、Drupal 等 11 种类型的博客。安装后进去第一个界面就是日志编辑，在未设置博客之前只能保存了。界面很简洁。

支持 Normal、Source、Preview 三种视图，并且能分别编辑日志、More 之前的内容和摘要，都支持 html。
支持标签、自定义字段和日志密码设置
支持是否允许评论，允许 Ping 的设置，可以更改日志的发布日期
只能以 FTP 方式上传日志中的图片
如果日志类别名称中有中文，那么很不幸，无法获取到
不支持日志的 Slug 名称的设置
本地的日志管理功能很弱

对于我来说，直接 Pass 掉。

2. BlogJet 星级 3.5

它是个商业版的软件，在 http://www.codingrobots.com/blogjet/ 下载，当前版本 2.5.0.15，大小 4.7M，30 天试用期，你要是不畏艰辛的能找到破解，都不在话下。

它的功能也较实，对于中文支持的很好，整体要优于前面的 BlogDesk。使用 WordPress 的接口来上传图片和附件，会上传到 wp-content/upload/ 中某个年月对应的目录中，还支持从 Flickr 中引入图片，没有 FTP 的支持。可编辑本地发布到的日志，如果在别处修改过此日志，再编辑就成问题了。无法设置日志的 Slug，即永久链接。又不能满足我的要求的。也缺乏本地日志管理功能。

3. Chrysanth WebStory 星级 1.5

商业软件，可试用 90 天，到 http://webstory.my/downloads/ 下载，个头还比较大，10.9M，功能看起来很齐全，还引入了本地数据库的管理。创建帐户时直接选择项里只有 Blogger、LiveJournal、ModBlog 和 Typepad 四个，但在定制 Provider 时有 20 多个博客引擎的选择，选 WordPress 就行。

不可否认，它看起来很强大，拥有像 WordPress 一样的编辑能力，但强大的让人不知所措，光设置博客就费我不少脑子，然而翻来翻去，真没找到要到哪去获取博客的类别，出来了，原来是自动获得的。中文类别会有问题，感觉对 WordPress 来说不太亲近，使用起来并不方便。发了一篇，贴了个图，点击发布，状态栏不停的报错，然后程序自己关闭了，不知道发生了什么。进到网站上看，文章有了，不过内容有问题，图片也没上来。

原来商业软件也不过如此啊，又难用还不能很好的满足需求。

4. Microsoft Word 2007 星级 2

一直都可以信手拈来的 WordPress 客户端都没有很好去重视，它很认同 WordPress，配置时可直接选择 WordPress 类型的博客。可以充分利用 Word 强大的编辑功能。

可选择类型，图片等文件也是使用 WordPress 的接口上传到 wp-content/upload 中的某目录，不过不知为何贴一个图片却给我上传了四个不同分辨率的图片。生成的的代码也还简洁，没有 FrontPage 生成大量垃圾代码的诟病。但是特定到博客方面的设置都没有，譬如标签，是否允许评论等，还不能使用 HTML 源代码编辑器。功能太简单，日志管理功能也没有，实际中还是不太考虑它的。

5. Post2Blog 星级 3

当前版本 3.0.1，免费的，在 http://download.cnet.com/Bytescout-Post2Blog/3000-12945_4-10702915.html?tag=lst-0-1 下载，9.3M。

支持很数十种类型的博客站点，图片的上传方式支持 WordPress 的内置方式，FTP、Flickr、www.imageshark.us、www.picturelli.com。支持代理。日志的编辑功能也很强，体现在下拉菜单中，可以插入高亮的代码块，但不太可取，是显式的对代码着色的。

可管理 Ping 服务器，设置是否允许评论和 Ping/Trackback。可编辑摘要，加入附件，支持中文分类，同样也是不能设置日志的 Slug，像我那种自定义链接中用了 %postname% 就不行了。还有不足之处就是编辑窗口太小，即使程序窗口全屏后，编辑窗口也保持原大小。

有谁知道我输入了多少个字，输入的是什么字。这个客户端还是不是的出现程序错误而关闭或失去响应，实在是不好对它多加评论，看上去不错，用起来问题多多。

没有自己的本地日志管理方式，居然发布前要求保存成一个个零散的 html 文件。本来印象还算不错，但最后想测试的时候，居然它与我的中文输入法搜狗五笔不兼容，不光打字困难，而且输入的内容显示出来还乱码，图片也没有成功的上传。直接对它的印象大打折扣，发布之后也是乱码，看来它对中文日志内容支持并不怎么好，降为 3 星。

本篇博客是使用开源的 Zoundry Raven 写的，个人比较推崇这个，也支持期盼已久的日志 Slug(缩略名) 的设置。
