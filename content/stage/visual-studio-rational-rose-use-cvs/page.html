Eclipse 内置就对 CVS 很好的支持，而 Visual Studio(.net) 和 Rational Rose 等对 VSS 支持很好，因为他们遵循的是 Microsoft SCC API，那么如何让 Microsoft SCC API 与 CVS 桥接起来呢？这就是现在要介绍的几个工具

1. Jalindi　这个程序完全免费，任你使用

Jalindi 的网址
http://www.devhood.com/tools/tool_details.aspx?tool_id=491
或
http://www.jalindi.com/igloo/index.html

Jalindi 让你在 Visual Studio(.net) 连接 CVS 的操作与 VSS 是一致的，是 Check Out/Check In，而不是像通常的 Update/Commit

2. CVS SCC Proxy  这是一个商业版的软件

CVS SCC Proxy 的网址是  < A href="http://www.pushok.com/soft_cvs.php">http://www.pushok.com/soft_cvs.php
可以在其网站上下载一个来试用一下，它是 CVS Plugin 的一种，它的功能比 Jalindi 要丰富一些，发现一个有意思的东西是 CVS SCC Proxy 有一个 Switch SCC 的工具，启动后在系统栏里，它居然能识别出我之前安装的 Jdlindi
也就是因其是一家商业公司，所以还提供了一些强劲的工具，如 SVN SCC Plugin、Power Merge Tool 等
不过你可以在网上充分发挥你的搜寻能力找到一个破解版来用。
在 Visual Studio(.net) 中操作跟 VSS 无多大差别，我在 Rational Rose 中试用了一下，感觉比较慢。
