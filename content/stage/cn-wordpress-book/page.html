拿回来几天时光一字不落的就把它给看完了，顺便记下些对我来说有意义的东西。被略过的内容并非就是它的不足，毕竟它的读者是广大的不同层次的博主们。

对我来说前面四章的内容用处不大，除第四章说到 WordPress 安装完后，wp-config.php 文件中最后配置一下 secret-key，它对于 cookie 中保存加密的用户数据增强了安全性。WordPress 3.0 之前是四个值，访问 http://api.wordpress.org/secret-key/1.1/，3.0 之后有八个值，访问 https://api.wordpress.org/secret-key/1.1/salt/ 获得后填入到 wp-config.php 文件中。要让界面（包括插件的）显示成中文，设置 define('WPLANG', 'zh_CN')。

Sidebar 除了放一些 WordPress 插件里的 Widget 外，我们还可以去到 http://www.widgetbox.com 找到一些好玩的游戏或其他功能，再借助于 Text Widget 来装扮一下。以后我们用得多了就会发现，像 Text Widget 这种最简单的却是最实用的。

安装插件 Ozh's Admin Drop Down Menu 可使得后台左边纵列的控制菜单横到上头去，下拉的方式，可节省不少空间。可是这一转换的过程应该也能看得到的，就这一变换过程可能不是那么的友好。

在媒体管理库中，每个附件可以关联到文章上去，用 Attach file 功能，这样可让你把握某个文件应用在了哪篇文章。不足之处，如果文件应用到了多个文件就无法表示出一对多的关系了。

文章记录可以分页显示，单篇文章内容也可分多页来展示，做法就是在分页处加上 <!--nextpage-->，想加多少个都行。如果你安装了 Advanced TinyMCE 插件的话，它默认会为你在 TinyMCE 编辑器工具栏上加个页面分页的按钮 Insert Page break。另外，如果要对页面分页建立目录导航做起来就稍为有点麻烦，还得找找有没有这方面的插件支持。

要对各类用户角色及相应的权限有较充分的了解。Administrator(管理员) 可不必说，主宰着一切；Subscriber(订阅者) 查看文章，且只能修改自己的资料；Author(作者) 可发表文章，但只能编辑自己的文章；Editor(编辑者) 可发表和修改所有的文章；Contributor(投稿者) 像 Author 一样可发表只编辑自己的文章，不过他们发表的文章必须经管理员审核后方可显示出来。

书中说在升级 WordPress 本身前应当禁用掉所有的插件，并使用默认的主题，完后再恢复现场，以免升级完后发现出错了。不过我觉得不必那么费事，不过前提是你对自己装的每一个插件要有一定程度的理解，并且升级前一定要有个完整的备份，还要有良好的紧急灾难应对处理的能力。

另外在升级时(也包括插件升级时)，因网速慢可能会遇到下载超时的情况，你可以改动这个超时时限，办法是找到 /wp-admin/include/file.php 中代码行：

$response = wp_remote_get($url, array('timeout' => 30)); 把其中的 30 秒改成你想要的大小。
